- feat: right-to-left support for the file editor (Monaco)

Extends the existing per-element direction detection to Cursor's file
editor. Each editor's visible lines are scored with the same
getMajorityDir() logic already used for chat/plan; RTL-dominant editors
are marked with a data-cursor-rtl-dir="rtl" attribute (a data attribute,
since Monaco rewrites the element's className).

Scoped CSS then:

- flows the view lines RTL and right-aligns them,
- overrides Monaco's per-line wrapper so isolated token

spans reorder right-to-left (leading list markers / punctuation / Latin
 runs land on the right instead of the left),

- moves the vertical scrollbar to the left edge.

Monaco has no native RTL cursor mode, so a focused-editor keydown handler
swaps the left/right arrows (preserving Shift/Alt/Ctrl/Meta) only while a
marked-RTL editor is active; LTR/code editors are never touched.

A new `cursorRtl.editorRtl` setting (auto | always | off, default auto)
and a "Editor Direction" command/quick-pick control the behaviour. The
mode is persisted to ~/.cursor-rtl-config.json; the loader injects it at
inject time and watches the file to push changes into open windows live,
so toggling applies without a reload.

README updated (Features, Commands, Settings, How It Works, Known
Limitations). Version left unchanged for the maintainer's release flow.

Co-authored-by: Cursor <cursoragent@cursor.com>

- docs: rename user-facing "file editor" to "Code Editor"
